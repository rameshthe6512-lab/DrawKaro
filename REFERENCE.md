## Architecture Overview

```
Browser A ──┐
Browser B ──┤─ WebSocket ─ drawkaro-server.js
Browser C ──┤─ Relays ops, chat, game state
Browser D ──┘─ WebRTC signaling
              └─ Peer-to-peer voice (WebRTC)
```

---

## Key Features Integrated

✅ **Drawing Sync** - All strokes broadcast in real-time
✅ **Chat** - Messages relay through server
✅ **Voice** - WebRTC peer connections for audio
✅ **Game Logic** - Synced game state across players
✅ **Replay** - Late joiners get drawing history
✅ **Cleanup** - Rooms auto-delete when empty

---

## Message Flow

```javascript
// Drawing Operation
Canvas → Net.sendOp() → Server → Broadcast → Other Canvases

// Chat Message
Input → Net.sendChat() → Server → Broadcast → All Chats

// Voice Audio
Mic → getUserMedia() → RTCPeerConnection → Other Players

// Game Start
Host → Net.startGame() → Server → Broadcast → All Clients
```

## Common Issues & Fixes

### Server won't start
```bash
# Wrong Node version?
node --version  # Must be 18+

# Port in use?
lsof -i :8080   # Check what's using port
kill -9 <PID>   # Kill if needed

# Missing ws?
npm install
```

### Can't connect to server
```bash
# Check server is running
npm start

# Check URL format
✓ ws://localhost:8080
✓ wss://domain.com
✗ http://localhost:8080

# Check firewall
# Must allow TCP 8080 (or your custom PORT)
```

### Deployment failed
```bash
# Check logs
heroku logs --tail
railway logs
render logs

# Check Procfile
cat Procfile
# Should be: web: node drawkaro-server.js

# Try rebuilding
git push heroku main
```
